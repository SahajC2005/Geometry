Geometric neural network pruning for time series

Applying the Separation Index compression method (Saffar et al. 2024) to 1D temporal models — train, compress, and measure efficiency on your dataset.

Separation Index (SI)

For every sample, check if its nearest neighbor in feature space belongs to the same class. SI = the fraction that do. Ranges from 0 (random) to 1 (perfectly separated).

Center-based SI (CSI)

Instead of nearest neighbor, check if each sample is closest to its own class centroid. Faster to compute, better correlated with accuracy. Used to size the FC layers.

Key insight for time series: This math works identically on 1D conv feature maps. A time series passes through 1D convolutions, producing feature vectors at each layer. SI measures their geometric separability the same way — no formula changes needed.
1
Remove ineffective layers

Plot SI at each layer's output. Find l* — the earliest layer where SI reaches within Pl% of SImax. Everything after l* gets cut.

For time series: pass your training data through the 1D-CNN. Compute SI at each conv layer output by flattening the temporal feature maps into vectors. The SI curve typically plateaus earlier in 1D models because temporal patterns are simpler than spatial ones.

(SI_max − SI(layer)) / SI_max × 100 ≤ Pₗ% (Pₗ = 1–2%)
2
Select effective filters

At layer l*, use forward selection to find the minimal set of filters that preserves SI. Greedily add the filter that most increases SI of the combined set.

Start with the single best filter. Iteratively add filters until adding k more doesn't improve SI by more than Pf%. This is where most parameters get pruned — often 60–75% of filters are redundant.

Pick filter set where SI(selected) ≈ SI(all filters) (within Pf%)
3
Design compact FC head

Replace the original FC layers with a minimal structure. Use CSI to size it — increase neurons until CSI at the FC output matches CSI at the pruned conv output.

CSI is faster to compute than SI (centroids vs nearest neighbors). Matching CSI between conv and FC output prevents over-parameterization. For time series, the FC head is typically very small — often just 2 hidden layers with 10–30 neurons.

(CSI(conv_out) − CSI(fc_out)) / CSI(conv_out) × 100 ≤ Pc% (Pc = 1%)
No retraining required. The pruned conv layers keep their original trained weights — only the small FC head needs training. This is a major advantage over methods like L1 pruning or lottery ticket that require full retraining after compression.

1
Start from your trained time series model

You don't retrain anything. SI works as a post-training analysis tool — it examines the features your model already produces.

Any 1D architecture works: ResNet-1D, InceptionTime, TCN, FCN, or even an LSTM with a convolutional front-end. You need your trained model + your labeled training dataset. That's it.

2
Plug SI into your model to diagnose each layer

Pass your training data through the model and extract the output (feature maps) at every convolutional layer.

Flatten each layer's feature maps into vectors — a 1D conv layer producing 64 filters of length 50 becomes a 3200-dimensional vector per sample.

Compute SI at each layer: for every sample, check if its nearest neighbor in that feature space shares its class label. The fraction that do = SI for that layer.

Plot SI across layers. You'll see it rise, then flatten. The flattening point tells you where your model stops learning useful features.

This is the diagnostic step — you're not changing anything yet, just measuring how much work each layer is actually doing on your time series data.
3
Compress: apply the 3-step algorithm

Cut layers: remove every layer past the SI plateau (l*). These layers burn compute but don't improve class separation on your data.

Select filters: at the last kept layer, use forward selection to find which filters actually contribute to SI. Often 60–75% of filters are redundant — they produce features that don't help separate your time series classes.

Refit FC head: replace the original fully connected layers with a tiny new one. Use CSI to size it — increase neurons until CSI of the FC output matches CSI of the conv output. This prevents over-fitting.

Only the small FC head needs training (typically seconds). The convolutional weights stay frozen from your original model.

4
Run both models on the same test data and compare

This is the efficiency test. You now have two models — original and compressed — and you run both on identical test data to measure every dimension of efficiency.

Why this tests efficiency on your dataset

The paper only tested SI on image data (CIFAR-10, ImageNet). By applying it to your time series dataset, you're answering a real open question: does geometric pruning work as well on temporal patterns as spatial ones? You run the exact same algorithm, on the same architecture class (1D convolutions instead of 2D), and compare parameter reduction, FLOP savings, and accuracy retention. If the compressed model achieves similar accuracy with significantly fewer parameters and faster inference — you've demonstrated that SI-based efficiency extends to time series. That's the contribution.
